Chapter 3 Images, Audio, and Video

图像：image

动画：animation

音频：audio

视频：video

学习内容：

  • 使用 \ 添加图像到页面、将图像变成链接、将一个图像分成多个部分并将不同的分本链接到不同的页面（图像映射，image map）
  • 研究网络上使用的主要图像格式（JPEG，GIF，PNG），这会影响 web 页面的加载速度
  • 更多的多媒体元素

1. 使用 \ 元素添加图片

必须有两个属性，src 属性指示路径，alt 属性写描述

<img src="logo.gif" alt="logo" />

\ 可以携带所有的通用属性和 UI 事件属性，以及

src alt align border height width hspace vspace ismap usemap longdesc name

src

指示 URL，可以是绝对或者相对

一般来说图像应该一直保持在自己的服务器上，别人无法移动

大多数网站作者为图像创建一个单独的目录

alt

必须出现在每个 \ 元素上并真实地描述图像，因为

  • 如果浏览器无法正确显示图像，可能显示文本
  • web 用户有时会使用屏幕阅读器，此时文本会描述他们看不到的图片
  • 有助于搜索引擎搜到自己的网站

有时图像不传达信息，只是装饰元素，此时 alt 属性仍然因该使用，但不赋值

<img src="xxxx" alt="" />

height / width

指定高度和宽度，以像素为单位

<img src="xxxx" alt="xxxx" height="120" width="180" />

属性的值也可以是浏览器屏幕的百分比，或者某一包含元素的百分比，即数字+百分号，但是百分比的情况非常少见

显示创建时的大小以外任何大小的图像都会引起图像失真或模糊

指定图像的大小是好习惯，应该尝试对所有图像使用这一属性，有助于页面更快加载（浏览器知道要分配多少空间），并且在加载的时候正确显示其它的部分

应该以创建与使用的图片大小相同的图片为目标，使用 Photoshop 等完成

  • 若页面上的图像比实际的大，会显示模糊

  • 若页面上的图像比实际的小，那么浏览器的加载速度会不必要地减慢，因为图像越小加载越快。同时，费流量费钱

在只指定了一个属性时，会自动保持长宽比，同时指定了两个属性但长宽比与初始不同时，会对图片进行伸缩

其它的属性很少使用/逐步弃用，在较早的网页上可看到

align，border，hspace，vspace，name 不建议

ismap，usemap

见之后的部分

longdesc

指示一个 URL，这个 URL 中有图片更详细的信息

有时图像会包含图形图标等非常多的信息，提供详细描述可以

  • 帮助搜索引擎
  • 帮助视觉障碍的用户

但是，任何主要浏览器都不支持 longdesc 属性，因此可以在图像旁边放一个链接进行详细描述。

2. 使用图像作为链接

<a href="../index.html" title="click here to return to home page">
    <img src="images/logo.gif" width="338" height="79" alt="logo" />
</a>

3. Image maps 图像映射

图像映射允许向同一图像添加多个链接，每个链接指向不同的页面。当图像被分成不规则区域时这个很有用，比如地图。每个可点击的区域称为 hotspot 热点

图像映射的工作方式基于 x 和 y 坐标，这些坐标从图像的左上角测量，有两种用法

  • 指定热点的位置
  • 确定用户点击图像的位置

通过比较热点的坐标和用户单击的坐标，可以将用户发送到该热点指定的链接

有两种类型的图像映射，作用相同。不同之处在于如何计算出点击的热点

  • Client-side image maps：web 浏览器计算出用户点击了哪一部分，并将用户带到相应页面
  • Server-side image maps：浏览器向服务器发送坐标，表示用户点击了图像的哪个部分，这些坐标由服务器上的脚本处理，该脚本决定用户应该被带到哪个页面

4. Choosing the Right Image Format

选择正确的图像格式保证网页加载不会不必要的变慢

网上大部分的图像的都是 bitmapped images 位图，将图像分成 pixels 像素网格并单独上色

图像每平方英寸的像素数称为图像的 resolution 分辨率，通常以每英寸 72 像素的分辨率保存在 Web 上使用的图像是正常的，因为，这对应着计算机屏幕上每平方英寸的像素数。相比之下，用于打印的图像提供给打印机时通常是 300 点每英寸。

图像每英寸包含的像素越多，文件的大小就越大，通过 Web 传播的时间就越长。因此，在网络上使用的任何图像都应以每英寸 72 点的分辨率保存，如果保存得更大，则会创建不必要的大文件，将花费更长的时间下载。

300 -> 72 很轻松，但是 72 -> 300 很难。拥有 300 的高分辨率图像时应该将 72 的在网络上使用，同时保留原始版本

浏览器倾向于支持三种常见的位图图像格式：

  • GIF：Graphics Interchange Format
  • JPEG：Joint Photographic Experts Group Format
  • PNG：Portable Network Graphics
